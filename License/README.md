## OMA LwM2M Object Licenses

Each **LwM2M Object** published in the OMNA LwM2M Registry **MUST include** a valid software license declaration. This ensures transparency, interoperability, and proper usage rights within the OMA and broader IoT communities.

Currently, the registry supports three license types:

**1. BSD-3-Clause License**

 - SPDX ID: BSD-3-Clause

- This is the **only license accepted** by the **Semantic Definition Format (SDF) for Data and Interactions of Things**, which standardizes object and model definitions across multiple Standards Development Organizations (SDOs).

- Recommended for broad interoperability and reuse.

**2. OMA BSD-3 License**

- Specific variant of the BSD-3-Clause license **used exclusively by OMA Working Groups** for official OMA-defined Objects.

- Ensures consistency across OMA-managed specifications.

**3. Custom License**

- Contributors may include their own license text; however, the use of non-standard or uncommon licenses (e.g., MIT, Apache, etc.) is **strongly discouraged**.

- Objects using such licenses will not be registered with **SDF** and are unlikely to be reused by other community members.

> Tip: To maximize interoperability and community adoption, OMA strongly recommends using the standard BSD-3-Clause license whenever possible.
> For further information, please refer to the [LwM2M Editor Guidelines](https://github.com/OpenMobileAlliance/lwm2m-registry/wiki/Guidelines)
